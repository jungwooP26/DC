# 0. 시험 직전 공식집

## 0.1 전송률·신호·지연

| 항목 | 공식/정의 | 기억할 점 |
|---|---|---|
| 데이터율 | \(N\) bps | 초당 전송되는 data bit 수 |
| 신호율 | \(S\) baud | 초당 signal element 수 |
| 신호당 비트 수 | \(r=\log_2 L\) | \(L\): 신호 레벨 수 |
| 데이터율-신호율 | \(N=S\times r\), \(S=N/r\) | 4 bit/signal이면 1000 baud = 4000 bps |
| 비트 지속시간 | \(T_b=1/N\) | 한 bit를 보내는 시간 |
| 대역폭-지연 곱 | \(B\times d\) | 링크 안에 동시에 존재할 수 있는 bit 양 |
| SNR | \(\mathrm{SNR}=P_{signal}/P_{noise}\) | 클수록 신호 품질이 좋음 |

## 0.2 최대 데이터율

### Nyquist: 잡음 없는 채널

\[
N_{\max}=2B\log_2 L
\]

- \(B\): bandwidth(Hz)
- \(L\): signal level 수
- 예: \(B=3000\text{ Hz}, L=2\)이면 \(N=2\times3000\times1=6000\text{ bps}\).

### Shannon: 잡음 있는 채널

\[
C=B\log_2(1+\mathrm{SNR})
\]

- SNR은 선형 비율을 사용한다.
- 예: \(B=3000\), \(\mathrm{SNR}=3162\)이면 \(C\approx34{,}860\text{ bps}\).
- SNR이 0이면 \(C=0\): bandwidth가 커도 신호를 복원할 수 없다.

## 0.3 PCM·TDM

- PCM 순서: **sampling → quantizing → encoding**.
- T-1:

\[
24\text{ channels}\times8\text{ bits}+1\text{ framing bit}=193\text{ bits/frame}
\]

\[
193\times8000=1.544\text{ Mbps}
\]

## 0.4 블록 코딩·Hamming distance

\[
n=k+r
\]

- \(k\): dataword bit 수
- \(r\): redundant bit 수
- \(n\): codeword bit 수

두 codeword의 Hamming distance는 서로 다른 bit 위치의 개수다.

| 목적 | 필요한 최소 거리 |
|---|---|
| 최대 \(s\)개 오류 검출 | \(d_{\min}=s+1\) |
| 최대 \(t\)개 오류 정정 | \(d_{\min}=2t+1\) |

## 0.5 CRC

1. dataword 뒤에 \(r\)개의 0을 붙인다. \(r\)은 generator의 차수다.
2. generator로 modulo-2 division(XOR 나눗셈)한다.
3. 나머지를 붙여 codeword를 만든다.
4. 수신 측에서 같은 generator로 나누어 syndrome이 0이면 오류 없음으로 판단한다.

\[
c(x)=d(x)x^r+\operatorname{remainder}\left(\frac{d(x)x^r}{g(x)}\right)
\]

### CRC의 대표 검출 조건

- generator가 두 항 이상이고 \(x^0\)의 계수가 1이면 모든 single-bit error를 검출한다.
- generator가 \(x^t+1\)을 나누지 못하면 거리 \(t\)인 isolated double-bit error를 검출한다.

### 표준 generator

| 이름 | 다항식 | 사용 |
|---|---|---|
| CRC-8 | \(x^8+x^2+x+1\) | ATM header |
| CRC-10 | \(x^{10}+x^9+x^5+x^4+x^2+1\) | ATM AAL |
| CRC-16 | \(x^{16}+x^{12}+x^5+1\) | HDLC |
| CRC-32 | \(x^{32}+x^{26}+x^{23}+x^{22}+x^{16}+x^{12}+x^{11}+x^{10}+x^8+x^7+x^5+x^4+x^2+x+1\) | LANs |

## 0.6 Checksum

### 전통적 checksum

1. 메시지를 동일한 길이의 word로 나눈다.
2. one's-complement addition으로 모두 더한다.
3. 합을 complement하여 checksum으로 보낸다.
4. 수신 측은 data와 checksum을 모두 더한 뒤 complement한다.
5. 결과가 0이면 수락한다.

### Fletcher checksum

\[
R=(R+D_i)\bmod256,\qquad L=(L+R)\bmod256
\]

\[
\mathrm{checksum}=L\times256+R
\]

### Adler checksum

\[
R=(R+D_i)\bmod65521,\qquad L=(L+R)\bmod65521
\]

\[
\mathrm{checksum}=L\times65536+R
\]

초깃값은 \(R=1, L=0\)이다.

## 0.7 ALOHA

| 방식 | vulnerable time | throughput | 최대 효율 |
|---|---:|---:|---:|
| Pure ALOHA | \(2T_{fr}\) | \(S=Ge^{-2G}\) | \(G=0.5\), \(S=1/(2e)\approx0.184\) |
| Slotted ALOHA | \(T_{fr}\) | \(S=Ge^{-G}\) | \(G=1\), \(S=1/e\approx0.368\) |

- \(G\): 한 frame time 동안 발생한 평균 전송 시도 수.
- Pure ALOHA는 임의 시점 전송, Slotted ALOHA는 slot 경계에서만 전송한다.

## 0.8 CSMA/CD 최소 프레임

충돌을 검출할 때까지 송신 중이어야 한다.

\[
T_{fr,\min}\ge2T_p
\]

\[
\text{minimum frame bits}=R\times2T_p
\]

예: \(R=10\text{ Mbps}, T_p=25.6\mu s\)이면 \(T_{fr}=51.2\mu s\), 최소 크기는 **512 bit = 64 byte**.

## 0.9 CDMA

- data bit 0 → \(-1\)
- data bit 1 → \(+1\)
- silence → 0
- 채널 합성 신호:

\[
D=d_1c_1+d_2c_2+\cdots+d_Nc_N
\]

- 송신자 \(i\)의 data 복원:

\[
d_i=\frac{D\cdot c_i}{N}
\]

Walsh code는 서로 직교하므로 \(c_i\cdot c_j=0\;(i\ne j)\), \(c_i\cdot c_i=N\).

## 0.10 Ethernet 효율

\[
\mathrm{Efficiency}=\frac{1}{1+6.4a},\qquad a=\frac{T_p}{T_{fr}}
\]

예: \(T_p=12.5\mu s\), \(T_{fr}=51.2\mu s\), \(a=0.24\)이면 효율은 약 **39%**.

## 0.11 다단 switch crosspoint

3-stage switch의 crosspoint 수:

\[
2kN+k\left(\frac{N}{n}\right)^2
\]

예: \(N=200,k=4,n=20\)이면 \(1600+400=2000\). 단일 200×200 crossbar의 40,000개보다 5% 수준이다.

# 1. Lecture 1: 데이터통신 개요와 계층 모델

## 1.1 데이터통신 기본

- **Telecommunication**: 먼 거리에서의 통신.
- **Data communication**: 합의된 형식의 데이터를 전송 매체를 통해 두 장치가 교환하는 것.
- 데이터는 수집된 값·사실이고, 정보는 데이터에 의미 또는 가치가 부여된 것이다.
- 데이터 형식: text, ASCII, Unicode, image, audio, video.

### 효과적인 통신의 4조건

1. **Delivery**: 정확한 목적지에 전달
2. **Accuracy**: 변형 없이 정확하게 전달
3. **Timeliness**: 필요한 시간 안에 전달
4. **Jitter**: packet 도착 시간 간격의 변동이 작아야 함

### 5가지 구성요소

`Message`, `Sender`, `Receiver`, `Medium`, `Protocol`

## 1.2 데이터통신 시스템 구성

| 장치 | 역할 |
|---|---|
| DTE(Data Terminal Equipment) | 사용자와 통신 시스템의 접점. 입력·출력·송수신 제어 |
| DCE(Data Circuit Equipment) | 단말/컴퓨터 신호와 통신 회선 신호 사이를 변환. MODEM, DSU, CODEC |
| CCU(Communication Control Unit) | 송수신 제어·감시, 문자 조립/분해, 오류 검출·정정, 회선/단말/자원 관리 |
| 통신 회선 | twisted pair, coaxial, optical fiber, satellite 등 |
| 컴퓨터 | 하드웨어와 통신 제어 프로그램을 통해 데이터 처리 |

- 지능형 단말: CPU·저장장치 내장, 독립 작업 가능.
- 비지능형 단말: 입력·출력만 수행, 독립 처리 불가.
- 역사: ARPANET(1969, 최초 유선 packet switching), ALOHA(1970, 최초 무선 radio packet switching).

## 1.3 전송 방향

| 방식 | 방향 |
|---|---|
| Simplex | 한 방향만 |
| Half-duplex | 양방향이지만 동시에 불가 |
| Full-duplex | 양방향 동시 가능 |

## 1.4 네트워크 평가 기준

- **Performance**: transit time, response time, throughput, delay
- **Reliability**: 실패 빈도, 복구 시간, 송수신 데이터 동일성
- **Security**: 비인가 접근 방지
- throughput/bandwidth는 초당 전송 가능한 bit 수, latency/delay는 송신부터 가능한 최초 수신까지의 시간이다.

## 1.5 Topology와 네트워크

- 물리 topology: 장치가 실제로 연결된 형태.
- 주요 형태: mesh, star, bus, ring, hybrid.
- 목표: **high throughput, low latency**.
- LAN: 좁은 지역, WAN: 넓은 지역. Internetwork는 LAN과 WAN의 결합.

## 1.6 Switching

### Circuit switching

- 통신 동안 dedicated path와 자원을 확보.
- `setup → data transfer → teardown`.
- 예: 전화망.
- 장점: 연결 후 일정한 전송, node별 지연이 작음.
- 단점: setup 지연, idle 상태에서도 자원 점유.

### Packet switching

- 메시지를 packet으로 나누어 node별로 **store-and-forward**.
- node에서 packet 전체를 수신·저장한 뒤 다음 node로 전달하므로 buffering과 queueing delay가 발생할 수 있다.
- 예: X.25, Frame Relay, ATM, Internet.

## 1.7 Internet 구조

- host/LAN → local ISP → regional ISP → backbone ISP.
- POP(Point of Presence): ISP 접속 지점.
- CPE(Customer Premises Equipment): 고객 구내의 modem 등.
- backbone ISP들은 NAP(Network Access Point)로 연결될 수 있다.

## 1.8 Protocol과 layering

- Protocol의 핵심:
  - **Syntax**: data block 형식
  - **Semantics**: 제어 정보, coordination, error handling
  - **Timing**: 속도 일치, 순서
- 복잡한 통신 작업을 layer별 subtask로 분리한다.
- 각 layer는 아래 layer의 서비스를 사용하고 위 layer에 서비스를 제공한다.
- 한 layer의 내부 구현이 바뀌어도 위 layer에 제공하는 service가 유지되면 다른 layer는 바꿀 필요가 없다.

# 2. Lecture 2: Physical Layer, 신호, 변환, 다중화

## 2.1 Analog와 Digital

- Analog signal은 연속적이며 주파수 스펙트럼이 제한될 수도 무한할 수도 있다.
- Digital signal은 이산적인 전압 pulse로 표현한다.
- digital signal의 장점: 저렴한 장비, noise 민감도 감소, attenuation 복원 용이.
- analog data/analog signal: 전화.
- analog data/digital signal: codec/PCM.
- digital data/analog signal: modem.
- digital data/digital signal: digital transmitter/line coding.

## 2.2 전송 방식

- **Baseband**: digital signal을 변조 없이 그대로 전송. low-pass channel.
- **Broadband**: digital data를 analog signal로 변조하여 band-pass channel로 전송.

## 2.3 Transmission impairment

| 손상 | 의미 |
|---|---|
| Attenuation | 이동하면서 에너지가 감소. amplifier로 보상 |
| Distortion | 주파수 성분별 전파 속도/지연 차이로 모양 변화 |
| Noise | thermal, induced, crosstalk, impulse noise |

## 2.4 Digital-to-digital conversion

### Line coding

- digital data를 digital signal로 변환.
- data rate와 signal rate(baud)를 구분한다.
- 분류:
  - Unipolar: NRZ
  - Polar: NRZ-L, NRZ-I, RZ, Manchester, Differential Manchester
  - Bipolar: AMI, pseudoternary
  - Multilevel: 2B1Q, 8B6T, 4D-PAM5
  - Multitransition: MLT-3

### Block coding

- \(m\) bit data block을 \(n\) bit codeword로 바꾸는 \(mB/nB\).
- redundancy를 추가하여 synchronization과 error detection을 개선.

### Scrambling

- 긴 0 sequence 등 synchronization을 깨뜨리는 pattern을 다른 pattern으로 치환.
- 대표: B8ZS, HDB3.

## 2.5 Analog-to-digital conversion

### PCM

`analog signal → sampling → quantizing → encoding → digital data`

- sampling: 일정 시간 간격으로 sample 추출
- quantizing: sample 값을 이산 level로 근사
- encoding: quantized level을 bit로 변환

### Delta Modulation

- 이전 sample과의 변화량을 추적.
- comparator, delay unit, staircase maker를 사용.

## 2.6 Transmission mode

| 구분 | 특성 |
|---|---|
| Parallel | 여러 bit를 동시에 여러 wire로 전송. 빠르지만 비싸고 거리 증가 시 skew |
| Serial | bit를 하나씩 전송. 저렴하고 장거리 적합 |
| Asynchronous | 문자마다 start/stop bit, 문자 사이 idle 가능 |
| Synchronous | frame 단위, start/stop bit overhead가 작음 |
| Isochronous | audio/video처럼 일정한 시간 간격을 보장 |

## 2.7 Digital-to-analog modulation

| 방식 | 바뀌는 carrier 성분 | 핵심 |
|---|---|---|
| ASK | amplitude | noise에 비교적 취약 |
| FSK | frequency | 각 data element에 다른 주파수 |
| PSK | phase | ASK/FSK보다 성능이 좋은 경우가 많음 |
| QAM | amplitude + phase | 더 많은 bit/signal element 가능 |

## 2.8 Analog-to-analog modulation

| 방식 | 변조 성분 |
|---|---|
| AM | carrier amplitude |
| FM | carrier frequency |
| PM | carrier phase |

## 2.9 Multiplexing

### FDM

- 각 신호를 서로 다른 frequency band에 배치.
- overlap 방지를 위해 guard band 사용.
- 예: 4 kHz 음성 3개를 guard band 없이 결합하면 12 kHz. 20~32 kHz 대역 사용.

### WDM

- optical fiber에서 서로 다른 wavelength를 결합.
- FDM의 광학적 형태.

### TDM

- digital 신호가 link의 시간을 나누어 사용.
- 각 source가 frame 안에서 time slot을 가짐.

| Service | Line | Rate(Mbps) | Voice channels |
|---|---|---:|---:|
| DS-1 | T-1 | 1.544 | 24 |
| DS-2 | T-2 | 6.312 | 96 |
| DS-3 | T-3 | 44.736 | 672 |
| DS-4 | T-4 | 274.176 | 4032 |
| E-1 | E-1 | 2.048 | 30 |
| E-2 | E-2 | 8.448 | 120 |
| E-3 | E-3 | 34.368 | 480 |
| E-4 | E-4 | 139.264 | 1920 |

## 2.10 Spread spectrum

- 필요한 최소 bandwidth보다 훨씬 넓은 bandwidth로 확산.
- eavesdropping과 malicious intruder에 대한 저항, 간섭 회피.
- FHSS: pseudorandom code에 따라 carrier frequency를 hop.
- DSSS: 각 data bit를 더 빠른 chip sequence로 확산.

| EHF | 30~300 GHz | LOS | radar, satellite |

- Radio wave: 대체로 omnidirectional.
- Microwave: unidirectional, line-of-sight, parabolic dish/horn antenna.
- Infrared: 300 GHz~400 THz, short-range, 벽 통과가 어려워 시스템 간 간섭이 적음.

## 3.3 Switching 3종

1. Circuit switching
2. Packet switching
   - Datagram
   - Virtual circuit
3. Message switching

## 3.4 Packet switching

### 장점

- link를 여러 packet이 동적으로 공유.
- 서로 다른 data rate의 station 연결 가능.
- circuit처럼 자원 부족으로 call 자체가 block되기보다 queueing delay 증가.
- priority 적용 가능.

### Datagram vs Virtual circuit

| 항목 | Datagram | Virtual circuit |
|---|---|---|
| setup | 없음 | 사전 경로 설정 |
| packet 경로 | 서로 다를 수 있음 | 모두 같은 경로 |
| packet header | destination address | VCI |
| 순서 | 뒤바뀔 수 있음 | 순서 보장 가능 |
| node 장애 | 우회 가능 | 해당 node를 지나는 VC 실패 |
| 예 | Internet | X.25, Frame Relay, ATM |

## 3.5 Switch 구조

- Circuit switch: space-division, time-division.
- Crossbar는 \(N\times N\)이면 \(N^2\) crosspoint.
- Multistage는 crosspoint를 크게 줄이지만 blocking 가능.
- Packet switch 구성: input port, output port, routing processor, switching fabric.
- Banyan switch는 \(N\log_2N/2\)개의 2×2 switching element를 사용하지만 collision 가능성이 커 batcher/trap module로 보완한다.

# 4. Lecture 4: Data-Link Addressing와 Error Control

## 4.1 Data-link layer

- 통신은 node-to-node.
- services: framing, flow control, error control, access control.
- broadcast link에서는 LLC와 MAC sublayer, point-to-point에서는 DLC sublayer 중심.

## 4.2 Link-layer address

- unicast, multicast, broadcast.
- Ethernet address는 48 bit(6 byte), 12개의 hex digit.
- broadcast: `FF:FF:FF:FF:FF:FF`.
- 첫 octet의 LSB(I/G bit):
  - 0: individual/unicast
  - 1: group/multicast

## 4.3 ARP

- IP address로 같은 link의 목적지 MAC address를 찾는다.
- request는 broadcast, reply는 보통 unicast.

| ARP field | 길이 |
|---|---:|
| Hardware type | 16 bit |
| Protocol type | 16 bit |
| Hardware length | 8 bit |
| Protocol length | 8 bit |
| Operation(request/reply) | 16 bit |
| Sender hardware/protocol address | 가변 |
| Target hardware/protocol address | 가변 |

## 4.4 오류 종류와 redundancy

- single-bit error: 한 bit 변화.
- burst error: 둘 이상의 bit가 일정 구간에서 손상.
- error detection은 오류 존재 여부, error correction은 손상 위치와 원래 bit까지 알아야 한다.
- redundancy는 data와 함께 extra bit를 보내는 핵심 원리다.

## 4.5 Linear block code

- valid codeword 집합을 미리 정의.
- 송신 codeword가 invalid codeword로 변하면 검출.
- linear code는 두 codeword의 XOR 결과도 valid codeword.
- simple parity check `C(5,4)`는 single-bit error는 검출하지만 짝수 개 오류는 놓칠 수 있다.

## 4.6 CRC 계산 절차

예: dataword `1001`, divisor `1011`.

1. divisor 차수 3이므로 `1001` 뒤에 `000`을 붙여 `1001000`.
2. XOR division.
3. remainder `110`.
4. codeword `1001110`.
5. 수신 codeword를 같은 divisor로 나누어 syndrome `000`이면 accept.

## 4.7 Forward Error Correction

- 실시간 multimedia에서는 재전송이 늦으므로 수신 측에서 즉시 복원.
- 방법: Hamming distance, XOR redundancy, chunk interleaving, combining, compounding.
- XOR:

\[
R=P_1\oplus P_2\oplus\cdots\oplus P_N
\]

손실 packet \(P_i\)는 나머지 packet과 \(R\)을 XOR해 복원.

# 5. Lecture 5: Data Link Control, HDLC, PPP

## 5.1 Framing

### Character-oriented

- Flag 사이에 header/data/trailer.
- data 안에 flag 또는 ESC가 나오면 **byte stuffing**으로 ESC 추가.

### Bit-oriented

- 대표 flag: `01111110`.
- data에서 1이 5개 연속이면 0을 삽입하는 **bit stuffing**.
- 수신 측은 연속 1 다섯 개 뒤의 0을 제거.

## 5.2 Flow control과 error control

- flow control: receiver가 처리 가능한 속도를 넘지 않게 조절.
- error control: 손상·손실 frame의 검출과 복구.
- FEC: 수신 측 자체 정정.
- BEC/ARQ: 송신 측에 오류를 알려 재전송.
- ARQ: Stop-and-Wait ARQ, Go-Back-N ARQ, Selective Repeat ARQ.

## 5.3 Simple protocol과 Stop-and-Wait

- Simple protocol: noiseless channel, receiver buffer 무한 가정, flow/error control 없음.
- Stop-and-Wait:
  1. frame 한 개 전송
  2. timer 시작
  3. ACK 대기
  4. timeout 또는 손상 ACK면 재전송
- sequence number가 없으면 ACK 손실 후 재전송된 frame을 duplicate로 전달할 수 있다.
- 0/1 sequence와 ACK number로 duplicate를 제거.
- Piggybacking: 양방향 통신에서 ACK를 반대 방향 data frame에 함께 실음.

## 5.4 HDLC

- bit-oriented protocol.
- point-to-point와 multipoint.
- transfer mode:
  - NRM(Normal Response Mode)
  - ABM(Asynchronous Balanced Mode)
- frame:

| 종류 | 용도 | 구성 |
|---|---|---|
| I-frame | user data | Flag, Address, Control, User information, FCS, Flag |
| S-frame | flow/error control | Flag, Address, Control, FCS, Flag |
| U-frame | link management | Flag, Address, Control, Management information, FCS, Flag |

- Control:
  - I-frame: `0`, \(N(S)\), P/F, \(N(R)\)
  - S-frame: `10`, code, P/F, \(N(R)\)
  - U-frame: `11`, 분할된 code, P/F

## 5.5 PPP

### PPP frame

| Field | 길이 |
|---|---:|
| Flag | 1 byte |
| Address | 1 byte |
| Control | 1 byte |
| Protocol | 1 또는 2 byte |
| Payload | variable |
| FCS | 2 또는 4 byte |
| Flag | 1 byte |

### Phase

`Dead → Establish → Authenticate → Network → Open(data transfer) → Terminate`

### Multiplexing protocol

- LCP: link establish/configuration
- AP: PAP, CHAP
- NCP: IPCP 등 network-layer 설정

| Protocol | 값 |
|---|---|
| IP data | `0x0021` |
| LCP | `0xC021` |
| PAP | `0xC023` |
| CHAP | `0xC223` |
| IPCP | `0x8021` |

### PAP vs CHAP

- PAP: authenticate-request → ACK/NAK, ID/password 전달.
- CHAP: challenge → response → success/failure, challenge-response 방식.

# 6. Lecture 6: Media Access Control

## 6.1 Multiple access 분류

| Random access | Controlled access | Channelization |
|---|---|---|
| ALOHA, CSMA, CSMA/CD, CSMA/CA | Reservation, Polling, Token passing | FDMA, TDMA, CDMA |

## 6.2 Pure/Slotted ALOHA

- Pure ALOHA:
  - 아무 때나 전송.
  - ACK가 없으면 random backoff 후 재전송.
  - vulnerable time \(2T_{fr}\).
- Slotted ALOHA:
  - frame time 크기의 slot.
  - slot boundary에서만 전송.
  - 중앙 clock/synchronization 필요.
  - vulnerable time \(T_{fr}\).

### 200-bit frame, 200 kbps 예제

\[
T_{fr}=200/200{,}000=1\text{ ms}
\]

Pure ALOHA:

| 발생량 | \(G\) | \(S=Ge^{-2G}\) | 성공 frame/s |
|---:|---:|---:|---:|
| 1000/s | 1 | 0.135 | 135 |
| 500/s | 0.5 | 0.184 | 92 |
| 250/s | 0.25 | 0.152 | 38 |

Slotted ALOHA:

| 발생량 | \(G\) | \(S=Ge^{-G}\) | 성공 frame/s |
|---:|---:|---:|---:|
| 1000/s | 1 | 0.368 | 368 |
| 500/s | 0.5 | 0.303 | 약 151 |
| 250/s | 0.25 | 0.195 | 약 49 |

## 6.3 CSMA

- “sense before transmit”, “listen before talk”.
- propagation delay 때문에 idle로 감지해도 collision 가능.

| 방식 | busy일 때 |
|---|---|
| 1-persistent | 계속 sensing, idle 즉시 전송 |
| nonpersistent | random time 대기 후 다시 sensing |
| p-persistent | slot마다 확률 \(p\)로 전송, \(1-p\)로 다음 slot 대기 |

## 6.4 CSMA/CD

- 송신하면서 medium을 monitor.
- collision 검출 시 transmission 중단, jamming signal, binary exponential backoff.
- 유선 shared Ethernet에서 사용.

## 6.5 CSMA/CA

- wireless는 송신 중 collision detection이 어려워 avoidance 사용.
- 핵심: interframe space, contention window, ACK.
- RTS/CTS로 hidden station 문제를 줄이고, 다른 station은 NAV 동안 대기.

## 6.6 Controlled access

| 방식 | 설명 |
|---|---|
| Reservation | reservation frame의 mini-slot으로 전송 권한 예약 |
| Polling | primary가 secondary를 select/poll |
| Token passing | token 보유 station만 전송 |

## 6.7 Channelization

- FDMA: bandwidth를 frequency band로 분할, 각 station에 고정 band.
- TDMA: time slot으로 분할, 각 station에 고정 slot.
- CDMA: 동일 시간·동일 bandwidth를 사용하되 직교 code로 구분.

### Walsh table 생성

\[
W_1=[+1]
\]

\[
W_{2N}=
\begin{bmatrix}
W_N & W_N\\
W_N & \overline{W_N}
\end{bmatrix}
\]

station이 90개면 \(2^m\ge90\)인 최소 \(m=7\), 즉 \(N=128\) sequence가 필요하다.

# 7. Lecture 7: Ethernet, Switch, VLAN

## 7.1 Ethernet 세대

| 세대 | 속도 |
|---|---:|
| Standard Ethernet | 10 Mbps |
| Fast Ethernet | 100 Mbps |
| Gigabit Ethernet | 1 Gbps |
| 10 Gigabit Ethernet | 10 Gbps |

IEEE 802는 LAN의 LLC와 MAC sublayer 및 physical layer 규격을 정의한다.

## 7.2 Standard Ethernet frame

| Field | 길이 |
|---|---:|
| Preamble | 7 byte, alternating 1/0 |
| SFD | 1 byte, `10101011` |
| Destination address | 6 byte |
| Source address | 6 byte |
| Type | 2 byte |
| Data + padding | 가변 |
| CRC | 4 byte |

- physical-layer header인 preamble+SFD를 제외한 Ethernet frame:
  - minimum 64 byte(512 bit)
  - maximum 1518 byte(12,144 bit)
- address는 byte 단위로 왼쪽→오른쪽, 각 byte 안에서는 bit를 오른쪽→왼쪽(LSB first) 전송.

## 7.3 Standard Ethernet 구현

| 구현 | 매체 | 최대 길이 | Encoding |
|---|---|---:|---|
| 10Base5 | thick coax | 500 m | Manchester |
| 10Base2 | thin coax | 185 m | Manchester |
| 10Base-T | 2 UTP | 100 m | Manchester |
| 10Base-F | 2 fiber | 2000 m | Manchester |

- shared bus/hub에서는 CSMA/CD와 collision domain.
- bridge/switch는 collision domain을 분리.
- full-duplex switched Ethernet에서는 collision이 없어 CSMA/CD가 필요 없다.

## 7.4 Fast Ethernet

- MAC frame format/address/minimum/maximum frame size는 유지.
- 10배 빨라졌으므로 같은 최소 frame을 유지하면 collision을 10배 빨리 검출해야 하며 network 길이를 줄여야 한다.

| 구현 | 매체 | 길이 | wire | Encoding |
|---|---|---:|---:|---|
| 100Base-TX | STP | 100 m | 2 | 4B/5B + MLT-3 |
| 100Base-FX | fiber | 185 m | 2 | 4B/5B + NRZ-I |
| 100Base-T4 | UTP | 100 m | 4 | 8B/6T |

## 7.5 Gigabit Ethernet

| 구현 | 매체 | 길이 | wire | Encoding |
|---|---|---:|---:|---|
| 1000Base-SX | short-wave fiber | 550 m | 2 | 8B/10B + NRZ |
| 1000Base-LX | long-wave fiber | 5000 m | 2 | 8B/10B + NRZ |
| 1000Base-CX | STP | 25 m | 2 | 8B/10B + NRZ |
| 1000Base-T | UTP | 100 m | 4 | 4D-PAM5 |

- half-duplex와 full-duplex가 정의되지만 대부분 full-duplex.

## 7.6 10 Gigabit Ethernet

- full-duplex only, CSMA/CD 미사용.
- LAN과 MAN까지 확장, IEEE 802.3ae.

| 구현 | 매체 | 길이 | wire | Encoding |
|---|---|---:|---:|---|
| 10GBase-SR | 850 nm fiber | 300 m | 2 | 64B/66B |
| 10GBase-LR | 1310 nm fiber | 10 km | 2 | 64B/66B |
| 10GBase-EW | 1350 nm fiber | 40 km | 2 | SONET |
| 10GBase-X4 | 1310 nm fiber | 300 m~10 km | 2 | 8B/10B |

## 7.7 Hub, switch, router

| 장치 | 동작 계층 | 특징 |
|---|---|---|
| Hub | Physical | signal 재생 후 모든 port로 전달 |
| Link-layer switch | Physical + Data Link | MAC address 확인, 선택 port로 forwarding |
| Router | Physical + Data Link + Network | IP 기반 network 간 routing |

## 7.8 Learning switch

1. frame의 source MAC과 들어온 port를 table에 학습.
2. destination이 table에 있으면 해당 port로만 forwarding.
3. 없으면 입력 port를 제외한 port로 flooding.
4. loop가 있으면 frame이 계속 복제될 수 있으므로 spanning tree로 일부 port를 blocking.

## 7.9 VLAN

- 물리 배선이 아니라 software로 구성한 logical LAN.
- membership 기준: interface/port number, MAC, IP, multicast address 또는 조합.
- 설정: manual, semi-automatic, automatic.
- multi-switch VLAN 정보 공유: table maintenance, frame tagging, TDM.
- VLAN은 broadcast domain을 논리적으로 분리한다.

# 8. 시험용 최종 비교

## 8.1 단위와 범위

| 계층/기술 | 전달 단위 | 범위 |
|---|---|---|
| Physical | bit | 한 hop |
| Data Link | frame | 한 hop |
| Network | packet | source host → destination host |
| Transport | segment/message | source process → destination process |

## 8.2 자주 섞이는 개념

| A | B | 구별 |
|---|---|---|
| bandwidth | throughput | 이론/채널 용량 vs 실제 달성량 |
| latency | jitter | 총 지연 vs 지연의 변동 |
| bit rate | baud rate | bit/s vs signal element/s |
| flow control | congestion control | 송수신자 속도 차 조절 vs network 혼잡 조절 |
| error detection | correction | 오류 존재 확인 vs 원래 data 복원 |
| FDM | FDMA | 다중화 기술 vs multiple access에 적용한 channelization |
| TDM | TDMA | 시간 다중화 vs station별 time access |
| CSMA/CD | CSMA/CA | 유선 collision detection vs 무선 collision avoidance |
| hub | switch | 모든 port 복제 vs MAC 기반 선택 전달 |
| datagram | virtual circuit | connectionless/경로 가변 vs setup/경로 고정 |

## 8.3 계산 문제 풀이 순서

1. 단위를 모두 bit, second, Hz로 맞춘다.
2. bit rate와 baud를 구분한다.
3. frame transmission time은 `frame bits / bit rate`.
4. propagation time은 `distance / propagation speed`.
5. ALOHA는 먼저 \(G=\lambda T_{fr}\)를 구한다.
6. CSMA/CD는 round-trip propagation \(2T_p\)를 사용한다.
7. CRC는 일반 나눗셈이 아니라 carry 없는 XOR division이다.
8. Hamming 문제는 “검출”인지 “정정”인지 먼저 확인한다.

```text
■ Whenever bits flow from one point to another, they
```

```text
12.1 Random Access
■ In random-access or contention-based, no station is
superior to another station and none is assigned
control over another. At each instance, a station that
has data to send uses a procedure defined by the
protocol to make a decision on whether or not to
send. This decision depends on the state of the
medium (idle or busy).
```

### Lecture_6 · PDF p.6

```text
12.1.1 ALOHA
■ ALOHA, the earliest random access method, was developed
at the University of Hawaii in early 1970. It was designed for
a radio (wireless) LAN, but it can be used on any shared
medium. It is obvious that there are potential collisions in this
arrangement. The medium is shared between the stations.
When a station sends data, another station may attempt to do
so at the same time. The data from the two stations collide and
become garbled.
```

### Lecture_6 · PDF p.7

```text
12.1.1 ALOHA (Cont.)
■ ALOHA protocol is developed for packet radio networks, but applicable to any
shared transmission medium.
□ A number of stations share the transmission medium. Two or more simultaneous
transmissions will cause a collision.
■ Sender
□ When station has a frame to send, it sends the frame
□ Station listens for a (fixed) amount of time
□ If it hears an ACK, fine(success).  If not, it retransmits the frame after a random time
□ If no ACK after several transmissions, it gives up
□ Frame check sequence can be used for error detection
■ Receiver
□ If the frame is OK and address matches receiver, sends ACK
□ Otherwise, ignores(discard) this frame and does nothing
■ Frame may be damaged by noise or by another station transmitting at the same
time (collision). Overlap of frames also causes collision.
■ ALOHA is simple, but very inefficient
□ Assuming random traffic, the maximum channel utilization is only about 18%
```

```text
12.1.2 CSMA
■ Why ALOHA and slotted ALOHA are so inefficient?
□ Stations don’t check the channel status.
(= = persistent mechanism 적용 않는다)
□ They just send out frames without considering whether the channel is
free or not, which creates too many collisions.
■ In fact, it is not difficult for a station to “sense” the channel
status (free or not). (유선 매체 환경에서는…, 반면 무선 매체

□ Stations listen to the (transmission) media (carrier sense)
□ Stations “know” whether the medium is idle(available) or not
□ Stations can transmit only if the medium is idle(free)
□ Collisions become rare
■ Only if two or more stations attempt to transmit at about the same time,
collisions could happen.
```

```text
13.3.1 Access Method
36
■ We remember that the proper operation of the CSMA/CD
depends on the transmission rate, the minimum size of the
frame, and the maximum network length. If we want to keep
the minimum size of the frame, the maximum length of the
network should be changed. In other words, if the minimum
frame size is still 512 bits, and it is transmitted 10 times faster,
the collision needs to be detected 10 times sooner, which
means the maximum length of the network should be 10 times
shorter (the propagation speed does not change).
```
