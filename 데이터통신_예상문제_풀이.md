## 1. 데이터 링크 계층의 주 기능과 기능 예

데이터 링크 계층은 **인접한 두 노드 사이에서 frame을 hop-to-hop으로 전달**한다.

| 주 기능 | 설명 | 기능 예 |
|---|---|---|
| Framing | 물리 계층의 bit stream을 구별 가능한 frame으로 나눔 | byte stuffing, bit stuffing, flag `01111110` |
| Physical addressing | 송신지·목적지의 link-layer 주소를 표시 | Ethernet MAC address |
| Flow control | 송신자가 수신자의 처리 속도를 넘지 않도록 조절 | Stop-and-Wait |
| Error control | 손상·손실 frame을 검출하고 복구 | CRC/FCS, ACK, timeout, ARQ |
| Access control | 여러 장치가 공유 매체를 사용하는 순서를 조절 | ALOHA, CSMA/CD, CSMA/CA |

대표 프로토콜은 **PPP, Ethernet(IEEE 802.3), Wireless LAN(IEEE 802.11)**이다.

## 2. Stop-and-Wait, Pure ALOHA, Slotted ALOHA, CSMA/CD 시간 계산

### 2.1 변수와 채점 기준

- \(T_{fr}\): data frame 전송시간
- \(T_p\): 편도 최대 전파시간
- \(T_{ack}\): ACK frame 전송시간
- \(T_{IFG}\): Ethernet interframe gap
- \(T_{jam}\): jamming signal 전송시간
- \(T_B\): random backoff 시간
- \(T_{TO}\): timeout 시간
- \(W_s\): 다음 slot 경계까지 기다리는 시간, \(0\le W_s<T_{fr}\)
- Slotted ALOHA의 slot 크기: \(T_{slot}=T_{fr}\)
- CSMA/CD의 slot time: \(2T_p\)

수치가 주어지지 않았으므로 답은 식으로 쓴다. 아래에서 ACK를 쓰는 방식의 성공시간은 **송신 시작부터 ACK가 송신자에게 돌아올 때까지**, CSMA/CD는 **마지막 bit가 목적지에 도착할 때까지**로 정의한다.

### 2.2 패킷 한 번 전송, 충돌 없이 성공

\[
A=T_{fr}+2T_p+T_{ack}
\]

| 프로토콜 | 성공시간 | 핵심 |
|---|---:|---|
| Stop-and-Wait | \(A\) | frame 전송 후 ACK를 받아야 다음 frame 전송 |
| Pure ALOHA | \(A\) | 즉시 전송, ACK가 없으면 실패로 판단 |
| Slotted ALOHA | \(W_s+A\) | 반드시 slot 경계에서 시작 |
| 1-persistent CSMA/CD | \(T_{fr}+T_p\) | idle이면 즉시 전송, Ethernet은 \(T_{fr}\ge2T_p\)여야 충돌 검출 가능 |

CSMA/CD에서 송신자 기준 전송 종료시간은 \(T_{fr}\)이고, 가장 먼 수신자에게 마지막 bit가 도착하는 시간은 \(T_{fr}+T_p\)이다.

### 2.3 패킷 두 개를 순차 전송, 충돌 없음

| 프로토콜 | 두 번째 패킷까지 성공시간 |
|---|---:|
| Stop-and-Wait | \(2A\) |
| Pure ALOHA | ACK를 확인하고 다음 frame을 보낸다면 \(2A\) |
| Slotted ALOHA | \(W_1+A+W_2+A\), 이상적으로 slot이 바로 맞으면 \(2A\) |
| 1-persistent CSMA/CD | 송신 완료 \(2T_{fr}+T_{IFG}\), 최종 수신 완료 \(2T_{fr}+T_{IFG}+T_p\) |

ACK와 전파시간을 무시하고 **순수한 채널 frame 시간만** 묻는다면 Pure/Slotted ALOHA는 두 frame이 \(2T_{fr}\), CSMA/CD는 \(2T_{fr}+T_{IFG}\)이다.

### 2.4 첫 번째 패킷이 한 번 충돌하고, 이후 재전송과 두 번째 패킷이 성공

충돌 후에는 timeout 또는 random backoff가 있으므로 **backoff 값이 주어지지 않으면 하나의 숫자로 결정할 수 없다.**

| 프로토콜 | 총 성공시간 |
|---|---:|
| Stop-and-Wait | \(F_{SW}+2A\) |
| Pure ALOHA | \(T_{fr}+2T_p+T_B+2A\) |
| Slotted ALOHA | \(T_{fr}+2T_p+T_B+W_r+A+W_2+A\) |
| 1-persistent CSMA/CD | \(T_{col}+T_{jam}+T_B+2T_{fr}+T_{IFG}+T_p\) |

- Stop-and-Wait에는 본래 공유매체의 충돌 처리 기능이 없으므로 충돌 frame을 **손실·손상 frame**으로 취급한다. timeout을 frame 전송 후부터 잰다면 \(F_{SW}=T_{fr}+T_{TO}\)이다.
- Pure ALOHA 강의 알고리즘: ACK를 \(2T_p\) 동안 기다린 뒤 실패하면 \(T_B\) 후 재전송한다.
- ALOHA backoff: \(T_B=R\times T_p\) 또는 \(R\times T_{fr}\), \(R\in[0,2^K-1]\).
- Slotted ALOHA에서는 backoff도 slot 단위이므로 보통 \(T_B=mT_{fr}\)로 둔다.
- CSMA/CD에서 \(T_{col}\le2T_p\), binary exponential backoff는

\[
T_B=R(2T_p),\qquad R\in[0,2^K-1]
\]

이다. 충돌을 검출하면 즉시 중단하고 jam signal을 보낸 뒤 backoff한다.

### 2.5 혼동하면 안 되는 시간

| 프로토콜 | 충돌 취약/검출 관련 시간 |
|---|---:|
| Pure ALOHA | vulnerable time \(=2T_{fr}\) |
| Slotted ALOHA | vulnerable time \(=T_{fr}\) |
| CSMA/CD | 최악의 충돌 검출시간 \(=2T_p\), 따라서 \(T_{fr,\min}\ge2T_p\) |

`vulnerable time`은 성공한 frame의 전체 ACK 왕복시간과 다른 개념이다.

## 3. PPP 인증방식

PPP는 `Dead → Establish(LCP) → Authenticate → Network(NCP) → Open → Terminate` 순서로 진행한다.

| 구분 | PAP | CHAP |
|---|---|---|
| 정식 명칭 | Password Authentication Protocol | Challenge Handshake Authentication Protocol |
| PPP Protocol 값 | `0xC023` | `0xC223` |
| handshake | 2-way | 3-way |
| 순서 | Authenticate-Request → ACK/NAK | Challenge → Response → Success/Failure |
| 전송 정보 | ID와 password | challenge와 shared secret으로 만든 hash response |
| password 자체 전송 | 전송함 | 전송하지 않음 |
| 재전송 공격 방어 | 약함 | 매번 다른 challenge를 사용하므로 강함 |
| 재인증 | 일반적으로 최초 1회 | 연결 중 다시 challenge 가능 |
| 장점 | 단순하고 구현이 쉬움 | PAP보다 안전함 |
| 단점 | 도청 시 ID/password 노출 가능 | shared secret과 hash 계산 필요 |

PAP packet code는 Request=`1`, ACK=`2`, NAK=`3`이고, CHAP code는 Challenge=`1`, Response=`2`, Success=`3`, Failure=`4`이다.

## 4. LAN A의 Host 1에서 LAN B의 Host 4로 전송

뒤의 loop 문제와 강의자료 Figure 17.5를 고려하여 다음과 같이 가정한다.

```text
LAN A: Host 1, Host 2 -- Hub A
                         |     |
                      Bridge 1 Bridge 2
                         |     |
LAN B: Host 3, Host 4 -- Hub B
```

각 bridge의 LAN A 측을 port 1, LAN B 측을 port 2라고 하고 초기 forwarding table은 비어 있다고 한다.

1. Host 1은 `source=MAC1`, `destination=MAC4`인 Ethernet frame을 만든다.
2. Hub A는 물리 계층 장치이므로 frame의 bit 신호를 Host 2, Bridge 1의 port 1, Bridge 2의 port 1로 모두 복제한다.
3. 두 bridge는 들어온 frame의 source를 보고 각각 `MAC1 → port 1`을 학습한다.
4. `MAC4`는 table에 없으므로 두 bridge 모두 입력 port를 제외한 port 2로 unknown-unicast flooding한다.
5. Hub B에는 같은 frame 두 개가 도착한다. Hub B는 각각을 Host 3, Host 4와 두 bridge 쪽으로 복제한다.
6. Host 3은 destination MAC이 다르므로 폐기하고, Host 4는 자기 주소이므로 받지만 같은 frame을 중복 수신할 수 있다.
7. 이때까지 bridge가 확실히 학습한 것은 source인 `MAC1`뿐이다. `MAC4`는 Host 4가 reply를 보내야 `MAC4 → port 2`로 학습된다.

정상적으로 bridge가 하나뿐이라면 reply 후 table은 다음과 같다.

| 학습 시점 | Bridge table |
|---|---|
| 초기 | empty |
| Host 1 → Host 4 | `MAC1 → port 1` |
| Host 4 → Host 1 reply | `MAC1 → port 1`, `MAC4 → port 2` |

---

## 5. 위 구조에서 loop problem이 발생하는 이유

bridge가 하나뿐이면 loop는 발생하지 않는다. **같은 두 LAN 사이에 Bridge 1과 Bridge 2라는 중복 경로가 있어 cycle이 만들어질 때** loop가 발생한다.

1. 목적지 MAC4가 unknown이므로 두 bridge가 모두 LAN B로 flooding한다.
2. 각 bridge가 내보낸 frame을 다른 bridge가 LAN B의 port 2에서 다시 받는다.
3. bridge는 source MAC1을 보고 기존 `MAC1 → port 1`을 `MAC1 → port 2`로 다시 학습하는 MAC flapping을 일으킨다.
4. MAC4는 여전히 unknown이므로 frame을 다시 LAN A로 flooding한다.
5. Ethernet frame에는 IP의 TTL과 같은 hop 제한 필드가 없으므로 frame이 스스로 사라지지 않는다.

결과는 **중복 frame, 잘못된 table 학습, broadcast/unknown-unicast storm, 대역폭 고갈**이다.

## 6. Spanning Tree로 loop 제거

Spanning Tree는 cable이나 port를 물리적으로 제거하는 것이 아니라, 중복 port 하나를 논리적으로 **blocking/discarding** 상태로 만든다.

### 두 bridge 예시

1. Bridge ID가 가장 작은 Bridge 1을 root bridge로 선출한다.
2. Root Bridge 1의 LAN A·LAN B port는 designated/forwarding port가 된다.
3. Bridge 2는 root까지 비용이 가장 작은 port 하나를 root port로 선택한다. 비용이 같으면 Bridge ID와 Port ID로 결정한다.
4. 예를 들어 Bridge 2의 port 1을 root port로 두고 port 2를 blocking한다.

```text
LAN A ===== Bridge 1 ===== LAN B
   \          root          /
    \-- Bridge 2 --X-------/
                   port 2 blocking
```

이제 Host 1의 frame은 Bridge 1을 통해 LAN B로 한 번만 전달된다. Bridge 2의 port 2는 data frame을 전달하지 않지만, 활성 경로 장애 시 STP 재계산 후 forwarding 상태로 바뀔 수 있다.

강의자료 Figure 17.8의 큰 예에서는 **S3의 port 2·3과 S5의 port 1이 blocking port**이다.

## 7. Protocol 정의 두 가지와 구성요소

강의자료에 나온 두 정의를 모두 적는다.

1. **일반적 정의:** 어떤 집단·조직·상황에서 받아들여지거나 확립된 절차 또는 행동 규범.
2. **컴퓨팅 정의:** 장치 사이에서 전자적으로 데이터를 교환하거나 전송하는 것을 지배하는 규칙의 집합.

즉, protocol은 통신 당사자들이 같은 작업을 수행하기 위해 따르는 **rules 또는 conventions의 집합**이다.

### 이번 학기 표현

| 요소 | 의미 |
|---|---|
| Syntax | data block의 형식, 필드 길이와 순서 |
| Semantics | 각 필드의 의미, 제어정보, 오류 처리 |
| Timing | 언제·어떤 속도·어떤 순서로 전송하는가 |

### 2학기 교재 표현

| 요소 | 의미 |
|---|---|
| Event | 어떤 사건이 발생했는가 |
| Action | 그 사건에 대해 무엇을 수행하는가 |
| Format | 주고받는 message/frame의 구조는 무엇인가 |

시험에서는 `Syntax·Semantics·Timing`과 `Event·Action·Format`을 모두 쓰는 것이 안전하다.

## 10. Nyquist Rate와 Shannon Capacity

### 10.1 Nyquist: 잡음 없는 채널

\[
N_{\max}=2B\log_2L
\]

- \(B\): bandwidth(Hz)
- \(L\): 신호 level 수
- \(N_{\max}\): 최대 bit rate(bps)

변형식:

\[
B=\frac{N}{2\log_2L},\qquad
L=2^{N/(2B)}
\]

예: \(B=3000\,Hz\), \(L=2\)

\[
N_{\max}=2(3000)\log_22=6000\,bps
\]

### 10.2 Shannon: 잡음 있는 채널

\[
C=B\log_2(1+\mathrm{SNR})
\]

SNR은 dB가 아닌 **선형 비율**을 넣는다.

\[
\mathrm{SNR}=10^{\mathrm{SNR}_{dB}/10}
\]

예: \(B=3000\,Hz\), \(\mathrm{SNR}=3162\)

\[
C=3000\log_2(1+3162)\approx34{,}860\,bps
\]

### 10.3 문제 풀이 순서

1. 채널이 noiseless이면 Nyquist, noisy이면 Shannon을 사용한다.
2. SNR이 dB로 주어지면 먼저 선형값으로 변환한다.
3. 실제 최대 전송률을 두 공식으로 모두 제한한다면

\[
R_{\max}\le\min\left(2B\log_2L,\;B\log_2(1+\mathrm{SNR})\right)
\]

4. 필요한 level 수는 \(L=2^{R/(2B)}\)로 구하고, 정수가 아니면 조건을 만족하도록 올림한다. 일반적인 digital modulation에서는 가능한 2의 거듭제곱 level로 올린다.

## 11. 시험 직전 핵심 문장

- Data Link: **frame, hop-to-hop, framing/MAC/flow/error/access control**.
- Pure ALOHA 취약시간 \(2T_{fr}\), Slotted ALOHA 취약시간 \(T_{fr}\).
- CSMA/CD: **sense → transmit while monitoring → collision → abort → jam → binary exponential backoff**.
- CSMA/CD 최소 frame: \(T_{fr}\ge2T_p\).
- PAP는 password 전송, CHAP는 challenge-response.
- Switch는 source를 학습하고 unknown destination을 flooding한다.
- Ethernet frame에는 TTL이 없으므로 중복 경로에서 loop가 지속될 수 있다.
- STP는 root를 정하고 중복 port를 blocking하여 loop 없는 tree를 만든다.
- Protocol: 두 정의와 함께 `Syntax/Semantics/Timing`, `Event/Action/Format`.
- Nyquist는 level 수를 포함하고, Shannon은 SNR을 포함한다.
